## 二、Step 19：LLM Bridge（OpenAI 大模型集成）

### 2.1 Rust 后端产出文件

| 文件 | 说明 |
|------|------|
| `src-tauri/src/llm/mod.rs` | LLM 模块入口（client / chat / prompts / retry） |
| `src-tauri/src/llm/client.rs` | OpenAI Client 初始化（环境变量读取 API Key + Base URL + Model） |
| `src-tauri/src/llm/chat.rs` | Chat Completions 封装（同步 + 流式）+ HTTP 请求骨架 |
| `src-tauri/src/llm/prompts.rs` | Prompt 模板管理（summarize / classify / enhance，版本化） |
| `src-tauri/src/llm/retry.rs` | 指数退避重试策略（1s→2s→4s，最多 3 次）+ 错误分类 |
| `src-tauri/src/commands/export.rs` | 结构化 Markdown 组装器（项目→MD，含时间轴/转录/OCR/标签/笔记） |
| `src-tauri/src/commands/llm.rs` | IPC 命令：get_llm_config / llm_summarize / llm_classify / llm_enhance_export |

### 2.2 前端产出文件

| 文件 | 说明 |
|------|------|
| `src/components/features/bridge/ExportPanel.tsx` | 导出预览面板（内容勾选 + Markdown 实时预览 + AI 润色 + 导出） |
| `src/components/features/bridge/TargetSelector.tsx` | LLM 目标选择器（NotebookLM / ChatGPT / Claude / 剪贴板） |
| `src/components/features/bridge/LLMSettingsForm.tsx` | LLM 设置表单（配置状态 + API Key 引导 + 离线降级说明） |
| `src/lib/ai/useLLMStream.ts` | Streaming 消费 Hook（Tauri Event 监听 + 打字机效果） |
| `src/lib/tauri-commands.ts` | 新增 6 个前端 IPC 封装函数 |

### 2.3 安全红线遵守

- [x] API Key 仅存在于 Rust 进程内存，从 `OPENAI_API_KEY` 环境变量读取
- [x] 前端永远不直接调用 OpenAI API
- [x] 日志中 API Key 仅显示前 8 位 `sk-xxxxx...`
- [x] API Key 未配置时 LLM 功能优雅灰置

### 2.4 架构设计

```
┌─── 前端 ──────────────────────────────────────┐
│ ExportPanel → invoke("export_project_markdown")│
│            → invoke("llm_enhance_export")      │
│            → navigator.clipboard / window.open │
│                                                │
│ LLMSettingsForm → invoke("get_llm_config")     │
│                                                │
│ useLLMStream → listen("llm-stream") Tauri Event│
└────────────────────────────────────────────────┘
           ↕ IPC
┌─── Rust 后端 ─────────────────────────────────┐
│ commands/export.rs → DB 查询 → Markdown 组装  │
│ commands/llm.rs → LLMClient → chat_completion │
│                                                │
│ llm/client.rs → env::var("OPENAI_API_KEY")    │
│ llm/prompts.rs → 集中化 Prompt 模板           │
│ llm/retry.rs → 指数退避（1s/2s/4s × 3 次）   │
└────────────────────────────────────────────────┘
```

## 三、Step 20：全局搜索 + 设置面板 + 应用优化

### 3.1 全局搜索

| 文件 | 说明 |
|------|------|
| `src/components/features/SearchPanel.tsx` | ⌘K 全局搜索面板（输入 + 结果列表 + 键盘导航 + 200ms 防抖） |
| `src/components/features/SearchResultItem.tsx` | 搜索结果项（类型图标 + 标题 + 高亮摘要 + 来源项目） |
| `src/hooks/useGlobalShortcuts.ts` | 全局快捷键 Hook（⌘K 搜索 / ⌘N 新建 / ⌘I Inspector） |
| `src/stores/searchStore.ts` | 更新：新增 `performSearch` 方法 |

**搜索架构**：前端 → searchStore.performSearch → IPC → Rust cmd::search → SQLite FTS5

### 3.2 设置面板

| 文件 | 说明 |
|------|------|
| `src/components/features/SettingsPanel.tsx` | 完整设置面板（6 个分类标签页 + ToggleSwitch 组件） |

**6 个设置分类**：

| 标签 | 配置项 |
|------|--------|
| 外观 | 主题（亮/暗/系统）、侧边栏宽度 |
| TF 卡 | 自动导入、导入后删除原文件 |
| 悬浮窗 | 启用/禁用、AI 分类、悬浮窗大小 |
| 音频 | 播放速度、Pre-roll 秒数、波形颜色、转录语言 |
| AI / LLM | API Key 配置（复用 LLMSettingsForm） |
| 隐私 | 匿名分析、数据存储路径、隐私声明 |

### 3.3 应用优化

| 优化项 | 实现方式 |
|--------|---------|
| React.lazy 按需加载 | SearchPanel / SettingsPanel 使用 `lazy()` + `Suspense` |
| 虚拟列表 | ProjectListView 已使用 `@tanstack/react-virtual` |
| 全局快捷键 | `useGlobalShortcuts` 集中管理 |
| 防抖搜索 | 200ms debounce |

### 3.4 集成更新

| 文件 | 变更说明 |
|------|---------|
| `src/App.tsx` | 重写：集成 SearchPanel + SettingsPanel + useGlobalShortcuts |
| `src/components/layout/AppLayout.tsx` | 新增 onSettingsOpen / onSearchOpen 回调 props |
