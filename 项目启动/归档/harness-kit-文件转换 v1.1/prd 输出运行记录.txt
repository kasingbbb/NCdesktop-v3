  录音格式的关键追问                                                                                                                     
                    
  录音转 MD 本质上是 ASR（自动语音识别），不是文档解析。有三种路径，每种都有成本：
                                                                                                                                           
  路径 A：macOS 原生 SFSpeechRecognizer                                                                                                    
  - 优点：无需外部 API，离线可用，项目已有 macos/ocr_ffi.rs 作为 FFI 先例                                                                  
  - 缺点：需要写 Swift/ObjC FFI bridge，工作量约 2-3 天；识别质量一般，长录音支持有限                                                      
                                                                                                                                         
  路径 B：调用配置好的 LLM API（如 Ark/OpenAI Whisper 端点）                                                                               
  - 优点：质量高，代码量少                                                                                                                 
  - 缺点：依赖 API Key 配置（用户没配 AI 就无法使用）；Ark API 不一定有音频转写端点；大文件传输慢                                          
                                                                                                                                           
  路径 C：引入 whisper.cpp 或类似本地推理库                                                                                                
  - 优点：离线、质量高                                                                                                                     
  - 缺点：依赖文件几百 MB，打包体积不可接受
