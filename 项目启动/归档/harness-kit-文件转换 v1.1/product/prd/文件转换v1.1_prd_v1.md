## 1. 项目概述

NCdesktop 是一款面向个人用户的本地知识管理桌面应用（Tauri + React，macOS）。P0 版本实现了文件拖入入库、后台内容提取（PDF/图片 OCR）、AI 分类打标等功能。

**v1.1 迭代目标**：将提取管线的输出从"数据库内部字段"升级为"工作区内可用的 .md 文件"，同时扩展多模态格式支持，覆盖 Word、PPT、录音等主流办公/记录格式。

### 核心体验变化

| 维度 | P0（现有） | v1.1（目标） |
|------|-----------|------------|
| 提取结果位置 | 数据库字段（仅 Inspector 可读） | 磁盘 .md 文件（工作区可见可用） |
| 支持格式 | PDF、图片 | PDF、图片、Word、PPT、录音 |
| 用户操作 | 拖入 → 等待 → 打开 Inspector → 读文字 | 拖入 → 等待 → 工作区出现 .md，直接用 |

## 2. 用户与核心场景

**目标用户**：使用 NCdesktop 管理学习/工作资料的 macOS 个人用户

**核心场景**：

| 场景 | 文件类型 | 期望结果 |
|------|---------|---------|
| 导入课程 PDF 讲义 | .pdf | 工作区出现 `讲义名.md`，可读课程文字内容 |
| 导入手写笔记照片 | .jpg/.heic | 工作区出现 `照片名.md`，OCR 识别的文字 |
| 导入 Word 作业草稿 | .docx | 工作区出现 `草稿名.md`，文档正文 |
| 导入 PPT 课件 | .pptx | 工作区出现 `课件名.md`，逐页幻灯片文字 |
| 导入课堂录音 | .mp3/.m4a | 工作区出现 `录音名.md`，语音转写文字 |

## 3. 功能需求（带优先级）

### P0 — 本版本必须

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| **物化写出 .md 文件** | 提取管线成功后，将 `structured_md` 写入 `<工作区>/<asset_id>_<stem>.md` | 工作区目录中有对应 .md 文件；文件内容非空 |
| **创建衍生 Asset** | 为写出的 .md 创建 `asset_type="markdown"`, `source_type="converted_from"`, `source_asset_id=原件ID` 的 Asset 记录 | DB 中存在关联记录；前端列表中出现新条目 |
| **DB 字段扩展** | `assets` 表增加 `source_asset_id TEXT` 可空字段 | migration 执行后原有数据正常；新衍生 Asset 的 source_asset_id 非空 |
| **Word (.docx) 提取器** | 解压 .docx zip → 解析 `word/document.xml` 中的 `<w:t>` 文字节点 → 输出 structured_md | 标准 .docx 文件提取后有可读文字；空文件 quality_level=0 |
| **PPT (.pptx) 提取器** | 解压 .pptx zip → 逐 slide 解析 `ppt/slides/slide*.xml` 中的 `<a:t>` 文字 → 每 slide 为 `##` 二级标题节 | 多页 .pptx 提取后文字按页分节；空 PPT quality_level=0 |
| **录音 ASR 提取器** | macOS SFSpeechRecognizer FFI bridge → 语音转文字 → structured_md | mp3/m4a/wav 在 macOS 上能触发转写；非 macOS 构建 can_handle 返回 false |
| **dropzone MIME 映射扩展** | `path_asset_meta` 覆盖 .docx/.pptx/audio 格式 → 正确设置 asset_type 和 mime_type | 上述格式拖入后 asset_type 非 "other"；进入提取管线 |
| **前端工作区来源标记** | 衍生 MD Asset 在 `AssetListView` 右栏显示"转换自 [原件名]"标记 | 用户能区分原件和衍生 MD；标记文字正确关联原件名 |
| **前端自动刷新** | `assetStore` 监听 `notecapt/asset-converted` 事件，收到后自动重新拉取当前项目 Asset 列表 | 提取完成后无需手动刷新，MD Asset 自动出现 |

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 提取为后台异步，不阻塞 UI；大文件（>50MB PDF、>10min 录音）设超时保护，超时后 quality_level=0，不物化 |
| **可靠性** | 物化写出失败（磁盘满、权限错误）不中断主流程，降级为仅 warn 日志，不影响原件入库 |
| **平台兼容** | audio_asr 提取器在非 macOS 构建时无副作用（can_handle 永返 false） |
| **数据安全** | 所有操作本地，无网络上传；原始文件不修改、不删除 |
| **可扩展性** | 新提取器遵循 `Extractor` trait 模式，未来新格式按同模式添加 |

## 5. 技术约束

来源：`session_context.md`

1. Tauri v2 + Rust 后端，SQLite 单连接 Mutex 保护
2. macOS Vision Framework 已接入（OCR FFI 先例在 `macos/ocr_ffi.rs`）
3. macOS SFSpeechRecognizer 通过 Swift FFI bridge 接入，复用 `build.rs` Swift 编译模式
4. Word/PPT 解析依赖 `zip` crate + `quick-xml` crate（需加入 `Cargo.toml`）
5. 衍生 Asset 写入使用现有 `db::asset::insert` 接口
6. 前端事件命名空间：`notecapt/`，新增 `notecapt/asset-converted`

### 不可妥协的技术底线

1. 原始文件绝对不删除，MD 版是衍生物
2. quality_level = 0 或 structured_md 为空 → 不写出 .md，不创建衍生 Asset
3. audio extractor 的 can_handle 在非 macOS 下永返 false
4. Word/PPT 本版本仅提取文字，跳过嵌入图片/图表

### 已识别的高风险项

| 风险 | 来源 | 当前状态 | 缓解策略 |
|------|------|---------|---------|

### MVP 边界声明

**做什么**：
- 照片、PDF、Word、PPT、录音 → 工作区自动出现对应 .md 文件
- 衍生 Asset 有来源关联（source_asset_id）
- 前端工作区展示"转换自"来源标记
- 前端事件驱动自动刷新，零干预

**不做什么**：
- 历史 Asset 回填（P1）：复杂度不合比例，不影响新增流程
- 音频转写质量优化（P2）：基础功能优先
- MD 手动编辑（P2）：需要独立编辑器组件
- 非 macOS 录音降级（P1）：当前仅 macOS 发布
- Word/PPT 图片提取（P2）：文字优先，图片解析复杂度高
