## 二、Step 15：时空记忆轴 — 关键帧轨道

### 2.1 产出文件

| 文件 | 说明 |
|------|------|
| `src/components/features/timeline/KeyframeThumb.tsx` | 关键帧缩略图组件（48px 默认→120px hover→80px 高亮） |
| `src/components/features/timeline/KeyframeConnector.tsx` | 缩略图到波形的虚线连接器（SVG 实现） |
| `src/components/features/timeline/KeyframeGroup.tsx` | 时间分组容器 + `groupKeyframesByTime()` 聚合算法（±2s 阈值） |
| `src/components/features/timeline/KeyframeTrack.tsx` | 关键帧轨道主容器（时间→X 坐标映射 + 拖拽/拖放集成） |
| `src/components/features/timeline/KeyframeContextMenu.tsx` | 右键菜单（查看详情/编辑备注/取消锚定/删除） |
| `src/hooks/useKeyframeDrag.ts` | 关键帧水平拖拽调整锚定时间 |
| `src/hooks/useKeyframeDrop.ts` | 从素材列表拖放创建新关键帧 |
| `src/components/features/timeline/TimelineView.tsx` | 更新：集成 KeyframeTrack |

### 2.2 技术要点

- **时间→像素映射**：`timeToX()` 基于 viewport 的 startTime/endTime 和容器宽度进行线性映射
- **时间分组算法**：先按 anchorTime 排序，相邻 ≤2s 的关键帧聚合为一组，水平排列
- **拖拽系统**：`useKeyframeDrag` 基于 mousedown/mousemove/mouseup 全局事件，计算像素偏移→时间偏移
- **拖放系统**：`useKeyframeDrop` 通过 `application/x-asset-id` DataTransfer 类型识别素材拖入
- **右键菜单**：固定定位 + 背景遮罩点击关闭，Lucide 图标

- [x] 关键帧按时间戳精准对齐到波形 X 轴位置
- [x] 缩略图 hover 平滑放大到 120×120px（duration-fast）
- [x] 关键帧可拖拽调整锚定时间

## 三、Step 16：AI 转录面板

### 3.1 产出文件

| 文件 | 说明 |
|------|------|
| `src/hooks/useTranscriptionSync.ts` | 基于 playback.currentTime 实时计算当前活跃转录段落 |
| `src/components/features/timeline/TranscriptionSegment.tsx` | 转录句子行（时间戳 + 说话人 + 文本 + 高亮 + 双击编辑） |
| `src/components/features/timeline/TranscriptionSearch.tsx` | 关键词搜索（匹配计数 + 上/下跳转 + 标黄高亮） |
| `src/components/features/timeline/TranscriptionPanel.tsx` | 转录面板容器（搜索栏 + 列表 + 自动滚动） |
| `src/lib/export/transcription-export.ts` | 转录导出工具（TXT / SRT / Markdown 三种格式） |

### 3.2 技术要点

- **实时同步**：`useTranscriptionSync` 订阅 `playback.currentTime`，反向遍历 segments 找到当前活跃段落
- **高亮机制**：活跃句子 → 品牌金色下划线 + 浅黄背景；搜索匹配 → `<mark>` 标黄
- **自动滚动**：播放状态下，`scrollIntoView({ behavior: "smooth", block: "center" })`
- **双击编辑**：切换 `<span>` → `<textarea>`，Enter 确认 / Escape 取消
- **搜索架构**：`TranscriptionSearch` 返回 `{ element, matches, currentMatchIndex }` 组合模式，由 Panel 集成
- **SRT 导出**：严格遵循 SRT 格式（序号、时间戳 `HH:MM:SS,mmm --> HH:MM:SS,mmm`、文本、空行）
- **Markdown 导出**：按说话人分组，引用块格式

- [x] 播放时当前句子高亮，自动滚动跟随
- [x] 点击任意句子，播放头精准跳转
- [x] 搜索关键词正确标黄匹配处
- [x] 双击可编辑修正文本
- [x] 导出 SRT 格式时间戳正确

### 4.2 技术架构

```
┌─────────── Magic Moment 数据流 ───────────┐
│                                            │
│  由图寻音（Image → Audio）：               │
│    KeyframeThumb.onClick                   │
│      → useMagicMoment.seekToKeyframe()     │
│        → timelineStore.seek(anchor - 5s)   │
│        → timelineStore.play()              │
│        → uiStore.magicMoment.isAnimating   │
│        → Playhead 平滑滑动动画             │
│        → uiStore.previewAssetId 更新       │
│        → ContentArea 切换素材预览          │
│                                            │
│  随音现图（Audio → Image）：               │
│    playback.currentTime 变化               │
│      → useMagicMoment.useEffect 检测       │
│        → |currentTime - anchorTime| < 150ms│
│        → uiStore.highlightedKeyframeId     │
│        → KeyframeThumb 放大到 80px         │
│        → magic-pulse 边框高光脉冲         │
│        → uiStore.previewAssetId 更新       │
│        → ContentArea 切换素材预览          │
│        → 2 秒后自动清除高亮               │
│                                            │
└────────────────────────────────────────────┘
```

### 4.3 Magic Moment 状态模型

```typescript
interface MagicMomentState {
  activeKeyframeId: string | null;     // 由图寻音：用户主动点击的关键帧
  highlightedKeyframeId: string | null; // 随音现图：播放经过时高亮的关键帧
  previewAssetId: string | null;        // 内容区联动：当前应预览的素材
  isAnimating: boolean;                 // 动效状态标记
}
```

### 4.4 动效规范

| 动效 | 参数 |
|------|------|
| Playhead 平滑滑动 | `transition: left var(--duration-normal) var(--ease-out-expo)` |
| KeyframeThumb 放大 | 48px → 80px，`var(--duration-fast) var(--ease-out-expo)` |
| 金色边框脉冲 | `magic-pulse` 动画 1.5s infinite |
| Playhead 指示器放大 | 8px → 12px + 金色光晕 `box-shadow` |
| 高亮持续时间 | 2000ms 后自动清除 |
| Pre-roll 倒退 | 默认 5 秒，设置可配 |
| 触发阈值 | |currentTime - anchorTime| < 150ms |
