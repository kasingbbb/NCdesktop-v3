## 2. 目标与范围回顾

### 2.1 解决的问题

1. **入口可见性**：无课日时侧边栏课程区不再整段消失；新增 **Calendar** 侧边栏项，始终可进入周视图。  
2. **浏览能力**：ContentArea 提供 **周一～周日时间网格**（8:00–22:00），支持周切换与点击课程进入预习空间。  
3. **回退一致性**：预习空间 **Back** 根据进入路径显示「← 课程日历」或「← 返回」，并恢复对应侧边栏分区与周起始日。

### 2.2 约束（已遵守）

- **后端不改动**：仅复用 `get_course_events` 时间范围查询。  
- **无新 npm 依赖**：周视图为 React + Tailwind + 现有 Zustand。  
- **Liquid Glass / CSS 变量**：沿用现有设计 token。

### 2.3 明确未做（P1/P2）

- 课程重叠并列布局（MVP 叠加显示）。  
- 月视图、系统日历同步、提醒（见 PRD）。

## 3. 架构摘要

### 3.1 ADR（与 Architect 文档一致）

| ID | 决策 |
|----|------|
| ADR-001 | 时间网格：Flex + 列内 **absolute** 定位课程卡片（非 Canvas）。 |
| ADR-002 | 当前周：`calendarStore.activeWeekStart`（YYYY-MM-DD，周一），**非 URL**。 |
| ADR-003 | 回退：`uiStore.coursePreviewReturnTo: { section, weekStart? }`，**非 history 栈**。 |

### 3.2 状态流（简图）

```
Sidebar「Calendar」→ activeSidebarSection = "calendar" → ContentArea 渲染 CalendarWeekView
周视图点课程 → coursePreviewReturnTo = { section: "calendar", weekStart } → course_preview
Back → 恢复 section + weekStart（若有）→ 关闭预习
```

## 6. 构建与验证

### 6.1 本地运行

```bash
cd 项目启动/NCdesktop
pnpm tauri:dev
```

### 6.2 建议手测场景

1. 侧边栏 **Calendar** 始终可见；点击进入周视图。  
2. 左右切换周、「今天」回到当前周；网格内课程位置与时长大致一致。  
3. 周视图点课程 → 预习 → **← 课程日历** → 回到周视图且周不变。  
4. 侧边栏 Today 列表点课程 → 预习 → **← 返回** → 回到原 `SidebarSection`。  
5. 当日无课：CourseSection 显示 Today +「今天没有课程」。

### 6.3 静态检查

开发结项时 TypeScript/ESLint 对本次改动文件无新增报错（以 IDE / `pnpm check` 为准）。
