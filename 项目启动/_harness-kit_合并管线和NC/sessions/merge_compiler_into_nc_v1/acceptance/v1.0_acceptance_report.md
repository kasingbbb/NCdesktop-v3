## 1. 项目摘要

NCdesktop 集成 KnowledgeCompiler (KC) 为 P0：用户拖入文档 → markitdown 转 MD → **KC enrich**（AI 标签 + 摘要 + 段落锚点 + Q&A 对）→ 落地增强 MD + Inspector / DocumentViewer 前端渲染。覆盖正常路径 + 5 类失败兜底 + 力 retrigger + 队列 toast + Settings UI + DMG 打包。

**6 周完工**：
- Week 1（5 工作日，7 task）：DB v18 schema + failure_code 5 KC 枚举 + kc 模块骨架 + KcSettings 7 字段 + MockKcServer + KcStatusBadge + frontmatter 解析依赖
- Week 2（5 工作日，4 task）：KcClient + KcProcessManager + lifecycle integration + settings loader
- Week 3（5 工作日，5+1 task）：enrichment 5 类失败映射 + scheduler 注入（12 行内）+ frontmatter writer + OutputStage 防御 + db_meta + **TD-3 关闭**
- Week 4（5 工作日，6+1 task）：Inspector render + DocViewer render + KcSettingsForm + retrigger + queue toast + 3 Tauri commands + **ai_enabled 字段补丁**
- Week 5（5 工作日，3 task）：5 类失败注入测试 + 4+2 e2e 集成 + perf benchmark
- Week 6（5 工作日，3 task）：DMG packaging script + venv 剥离 + **kc_persist DRY 单源化**

**实际工作量**：30 工作日（贴 PRD §"不可妥协" #8 上限 35 工作日，含 PM ESCALATE 放宽的 5 日 buffer 未用尽）。

### 6.1 50 篇语料选取（5 来源 × 10 篇 = 50）

| 来源 | 数量 | 推荐选材 |
|------|------|----------|
| PDF 教科书 | 10 | 计算机 / 数学 / 物理 / 历史 各 2-3 章节，含图表 |
| DOCX 文档 | 10 | 工作报告 / 论文初稿 / 合同范本 / 政策文件 |
| 网页（HTML → PDF）| 10 | 技术博客 / 维基百科 / 新闻 / 政府公告 |
| TXT 笔记 | 10 | 个人笔记 / 会议记录 / 文章草稿 |
| Markdown 笔记 | 10 | 技术文档 / 项目 README / 个人 wiki |

### 6.2 评分维度（每篇 3 项 × 5 分 = 15 分满分；50 篇均分阈值在 §6.4）

1. **标签语义命中率**（5 分）：AI 标签是否真匹配文档主题？计算 = 命中标签数 / 总标签数 × 5
2. **问答来源命中率**（5 分）：Q&A 对是否真能从原文找到答案？计算 = 答案位置命中数 / 总 Q&A 数 × 5
3. **整体可读性提升感**（5 分）：增强 MD 比 markitdown 原版主观可读性提升多少？1=无 / 3=中等 / 5=显著

### 6.3 评分模板

```yaml
# acceptance/corpus_scoring.yaml（PM 填写）
samples:
  - id: "pdf_textbook_01"
    source: "PDF 教科书"
    file_path: "..."
    file_size_kb: ...
    kc_enriched: "true|partial|false"
    failure_code: null  # or E_KC_*
    tag_accuracy_score: 1-5
    qa_accuracy_score: 1-5
    readability_lift_score: 1-5
    notes: "..."
  # ... 49 more
```

### 6.4 通过阈值（PRD §6.6）

- **PASS**：3 维度均分 ≥ 3.5 / 5（即"中等以上"）
- **CONDITIONAL PASS**：任 1 维度均分 [3.0, 3.5)，需在 P1 迭代修复
- **FAIL**：任 1 维度均分 < 3.0 → 不准予上线
