## 1. 背景与目标

### 1.1 背景一句话

NoteCapt 已经在 0.1.0 跑通"拖入 → 提取 → 知识合成 → 镜子反馈"主管线。但**用户对 AI 输出的信任**仍未建立——AI 的引文不可点回原文、用户主动标注的内容被埋没、长录音难以精确召回、跨模态素材不能联动检索。这套问题在 r/notebooklm 1387 帖论坛分析中是排名最靠前的三大用户痛点，**社区已经手动总结出 60+ 条最佳实践（Master Index、源切分、Anti-Hallucination 系统提示、Glossary）**，但 NotebookLM 用户必须花 30 分钟手工配置才能享用。

### 1.2 本 PRD 要做的事一句话

**把这套最佳实践从"用户手工配置"升级为"NoteCapt 默认开启的产品行为"**，全部围绕一根承重柱：**给每段内容打 INDEX（可溯源的位置标签）**。

### 1.3 业务目标

| 目标 | 量化指标 |
|---|---|
| 让 AI 每句话都能溯源 | AI 输出中 ≥ 95% 的事实陈述附带可点击位置标签 |
| 让用户能 1 秒回原文核对 | 点击引用标签 → 跳转原文（PDF 页码 / 音频秒数 / 章节）首次响应 < 500ms |
| 让长文档 / 长录音的召回率显著提升 | 同样问题在 200+ 页 PDF / 30+ 分钟录音中的召回准确率 > 90% |
| 让用户主动标注的内容不被遗漏 | 包含用户高亮的内容在镜子反馈中的命中率 > 98% |
| 让跨模态素材能联动 | 同一时间窗的录音 + 板书 + 笔记，在相关问答中 ≥ 80% 联合召回 |

### 1.4 非目标（本 PRD 不解决）

- **不涉及**模型选型与训练（沿用现有 LLM）
- **不涉及**云同步与多端（NoteCapt 0.2-0.4 阶段仍为本地优先）
- **不涉及**协作 / 分享 / 社区功能
- **不涉及**MCP 反向暴露（已规划至 M4 之后）

## 2. 目标用户与场景

### 2.1 目标用户

| 类型 | 描述 | 优先级 |
|---|---|---|
| **学生 power user** | 长期使用 NotebookLM/Claude 处理学习材料，对幻觉/漏读痛深感强 | P0 |
| **研究人员** | 处理大量文献、对引用准确度有严苛要求 | P0 |
| **自学者 / 转行者** | 处理大量散乱资料，需要"系统替我消化"的体验 | P1 |
| **创作者 / 知识工作者** | 处理跨主题素材库，重视"我标过的别漏掉" | P1 |

### 2.2 核心使用场景

**场景 A · 阅读一份 200 页教材**
> 用户拖入《保险保障一生》PDF（280 页）。10 分钟后他问："保险的本质是什么？"  
> ✅ 期望：AI 回答附带 ⟨p.12-14⟩ 锚，点击跳转 PDF 对应页；用户对"AI 是否在编"无怀疑。

**场景 B · 复盘一节 30 分钟课**
> 用户当天录了一段 30 分钟讲座录音 + 拍了 3 张板书 + 课后写了笔记。一周后他对镜子反馈讲："我记得老师说保险有三个误区"。  
> ✅ 期望：AI 同时召回录音段 ⟨14:08-14:14⟩ + 当时拍的板书 + 笔记，告诉他"你提到了误区一、二，板书上还圈了误区三，你没讲到"。

**场景 C · 把整份课程喂给外部 AI**
> 用户想把 NoteCapt 处理过的 8 份课程文档喂给 ChatGPT 写一份综述。  
> ✅ 期望：拖入的每份 .md 自带"反幻觉前言 + 目录"，ChatGPT 立即按 NoteCapt 的约束回答，不需要用户再输 Custom Instructions。

**场景 D · 处理含大量表格的金融报告**
> 用户拖入一份含 20+ 张表格的金融研报。问："保险 A 和保险 B 哪个保额高？"  
> ✅ 期望：AI 能准确给出"保险 A 10 万、保险 B 15 万"——表格内容不再被漏读。

## 4. 核心功能需求详述

### F1 · 内容片段（Chunk）与位置标签 【P0 | M1】

#### 用户故事

> 作为 NoteCapt 用户，当我读到 AI 给我的任何一句回答时，我希望立刻能看到这句话来自我哪份文档的哪个具体位置，并能一键回到原文核对。

#### 功能描述

NoteCapt 在导入每份文档时，**自动把内容切分成可独立引用的"内容片段"**，并给每个片段挂上"位置标签"。位置标签的形式随文件格式不同：

| 文件格式 | 位置标签示例 |
|---|---|
| PDF | `⟨p.12-14⟩` |
| DOCX / Markdown | `⟨§3.1.2⟩`（章节路径）|
| PPTX | `⟨slide 7⟩` |
| 音频 | `⟨00:23:14-00:25:30⟩` |
| 图片 / 板书 OCR | `⟨左下区域⟩`（含 bbox 坐标）|
| 网页 / 邮件 | `⟨#section-3⟩` |
| CSV / 表格 | `⟨row 42, col 价格⟩` |

**用户视角**：
- 文档列表里仍是一份完整 .md 文件（不切散）
- 阅读 .md 时每个章节标题旁有低调的位置标签
- AI 回答里每句事实陈述都附带可点击的位置标签
- 点击位置标签 → 弹出原文上下文 / 跳转 PDF 对应页 / 跳转音频对应秒

**系统视角**：
- 内部建立"片段索引"数据库
- 所有后续 AI 调用都基于片段，不再直接面对完整长文

#### 验收标准（DoD）

- [ ] 导入 PDF 后，每章节都生成片段记录（基于 heading / TOC）
- [ ] 导入音频后，按语义段（非固定时长）生成片段记录
- [ ] AI 回答（摘要、镜子反馈、问答）每条事实陈述都附带位置标签
- [ ] 点击位置标签能跳转到原文对应位置（PDF / 音频 / 图片）
- [ ] 在 Inspector 详情面板可查看该文件的片段统计（共 N 个片段、按章节分布）

#### 物理形态参考

详见《物理形态规范 v1.0》§3。本 PRD 不重复细节。

### F2 · 反幻觉协议前言（Grounded Prompt Preamble）【P0 | M1】

#### 用户故事

> 作为 NoteCapt 用户，当我把 NoteCapt 处理过的 .md 文件拖到 ChatGPT 或 Claude 时，我希望对方 AI **立刻知道游戏规则**——只引用本文档、不要补全、找不到要说找不到——我不需要再手动复制 Custom Instructions。

#### 功能描述

每份 NoteCapt 产出的 .md 文件头部自动嵌入一段"AI 工作约束"前言。前言内容**从 NoteCapt 应用内的 AI Prompt 配置面板**（截图里那个）读取，由用户可改。

**前言示意**：

> 🔒 **NoteCapt Grounded Prompt**  
> 本文件由 NoteCapt 从「XXX」自动提取。回答时请：  
> 1. 只引用本文档内容；本文档以外的内容必须显式说"非本文档"  
> 2. 每条引用必须挂位置标签  
> 3. 找不到答案时直说"在本文档中未找到"——禁止补全  
> 4. 不要 "summarize"，请 "explain in detail with citations"  
> 5. 用户高亮 / 用户笔记是高权重信号

**NoteCapt 内部 AI 调用**：同样的前言注入到 prompt 拼装层，用户视角看不到。

#### 验收标准

- [ ] AI Prompt 配置面板新增 `grounded_prompt_preamble` 配置项，提供默认文案 + 用户可改
- [ ] 所有导出的 .md 头部都有该前言块
- [ ] NoteCapt 内部所有 AI 调用（摘要 / 镜子反馈 / 问答）都自动注入
- [ ] 用户改前言后，下次产出的文件立即生效；历史文件可右键"重新注入前言"
- [ ] 用户测试：把任意 .md 拖到 Claude → Claude 表现为"我只引用本文档" → 通过

#### 度量

| 指标 | 目标 |
|---|---|
| 拖到 Claude/ChatGPT 后对方 AI 主动遵守约束的比例 | > 90% |
| AI 内部调用中"找不到 → 编造"的失败率 | < 3% |

### F3 · 项目地图（Master Index）【P0 | M1-M2】

#### 用户故事

> 作为 NoteCapt 用户，我打开一个项目时希望立刻看到"这个项目里有什么"——所有文档清单、章节大纲、关键术语——不用挨个点开每个文件看。

#### 功能描述

每个项目自动生成一份用户可见的 `_master_index.md` 文件：
- 列出所有文档（含类型、大小、主题标签）
- 汇总所有文档的章节大纲
- 列出跨文档关键术语 + 出现位置
- 列出 NoteCapt 自动发现的"跨文档语义关联"（如"文档 1 §3 ↔ 文档 4 §2 相似度 0.83"）

**两种使用方式**：
1. 用户在 NoteCapt 工作区面板能看到这份文件，可打开查阅
2. NoteCapt 内部所有 AI 调用前**自动注入** Master Index 摘要到 prompt，给 AI 一张"全局地图"

**增量更新**：新文件入库时**增量追加**，不是重建（避免覆盖用户对该文件的人工修改）。

#### 验收标准

- [ ] 项目根目录自动生成 `_master_index.md`，首字符 `_` 保证文件列表排序在前
- [ ] 新文件入库 / 删除时自动增量更新，不破坏用户的手工修改区域
- [ ] AI 内部调用时该 Index 自动注入 prompt
- [ ] 跨文档语义关联探测在后台异步跑，不阻塞用户

### F4 · 项目术语表（Glossary）【P1 | M2】

#### 用户故事

> 作为 NoteCapt 用户，我希望 NoteCapt 自动认得我学习领域的专业术语，并且我能花 30 秒帮它补全我觉得重要的定义——下次 AI 召回的准确度立刻提升。

#### 功能描述

NoteCapt 在导入文档时**自动抽取**专有名词、缩写、高频名词，进入"待定义术语队列"。

**用户可见层**：
- 项目根目录有 `_glossary.md` 文件，可打开查看 / 编辑
- "今天"页空闲时弹微任务："花 30 秒帮 AI 理解'杠杆率'" → 用户写 / 不写都可以

**系统层**：
- 术语表内容自动注入到 AI prompt
- 召回时给"匹配到术语"的片段权重 boost

#### 验收标准

- [ ] 导入文档后 24h 内自动生成 / 更新 `_glossary.md`
- [ ] 术语状态分三档：`候选 / 已确认 / 已忽略`
- [ ] 用户编辑后内容同步到数据库
- [ ] "今天"页能从候选队列弹微任务，30s 完成
- [ ] AI 召回准确率：有 glossary 加持的查询 vs 无 glossary 的 A/B 对比，准确率提升 ≥ 15%

### F5 · 用户标注信号 【P1 | M2】

#### 用户故事

> 作为 NoteCapt 用户，我在 PDF 上的高亮、批注、划线是我**主动告诉系统**这段内容重要——我希望 NoteCapt 不仅识别这些标注，还要在 AI 召回和镜子反馈时**优先关注**它们。

#### 功能描述

**抽取**：导入 PDF 时同时抽取 PDF 自带的 annotations（高亮 / 批注 / 下划线），保留在衍生 .md 中。

**召回 boost**：包含用户标注的片段，在检索时权重 ×1.5。

**镜子反馈专门提醒**：用户讲解时如果"自己曾经标注过的概念没讲到"，AI 主动提醒——"你在 ⟨p.12⟩ 标注过 X，你这次没提到"。

#### 验收标准

- [ ] PDF 自带高亮 / 批注被正确提取到衍生 .md
- [ ] 标注片段在检索结果中可见标识（"包含用户标注"）
- [ ] 召回时标注片段权重生效
- [ ] 镜子反馈中"用户标注未讲到"提醒触发率 > 80%（用真实 PDF + 高亮回归测试）

### F6 · 跨模态时间锚共召回（杀手锏功能）【P2 | M3】

#### 用户故事

> 作为 NoteCapt 用户，当我说"老师上次讲的"或"那天我笔记里写的"时，我希望 NoteCapt 同时召回那段录音、当时拍的板书、当天写的笔记——而不是只给我一段文字。

#### 功能描述

**自动时间关联**：同一天的素材按时间窗（默认 ±15 分钟）自动关联：
- 录音的录制时间（文件 metadata）
- 图片的 EXIF 拍摄时间
- 笔记的 created_at

**跨模态共召回**：当 retrieval 命中某个时间窗内的素材时，自动一起召回其他模态的素材。

**用户可见层**：
- 项目根目录 `_timelines/_timeline_YYYY-MM-DD.md` 文件汇总当天所有素材
- 当前的"时间流视图"（截图 2）从纯可视化升级为可点击进入时间线详情

**镜子反馈升级**：
> "你提到了录音里的误区一（[听原音] ⟨00:03:20⟩），但你拍的板书 ⟨14:14 参考截图.png⟩ 上还圈了误区三，你没讲到"

#### 验收标准

- [ ] 同一天 ≥ 2 个素材时，自动生成 `_timeline_YYYY-MM-DD.md`
- [ ] 时间窗内的不同模态素材自动建立关联
- [ ] 镜子反馈能输出跨模态联合反馈
- [ ] **杀手锏 demo**：录一段课 + 拍 3 张板书 + 写笔记，第二天问"老师讲的三误区" → AI 输出包含至少 2 个模态的引用

#### 度量

| 指标 | 目标 |
|---|---|
| 同一时间窗素材的联合召回率 | > 80% |
| 用户对该功能的"惊喜"程度（NPS-like）| > 60 |

### F7 · 召回透明化 UI 【P1 | M2-M3】

#### 用户故事

> 作为 NoteCapt 用户，当 AI 给我一个回答时，我希望能看到它"读了我哪几段"——这样我能判断它是不是真的看了，有没有漏读关键内容。

#### 功能描述

**主要交互**：知识单元页 / 镜子反馈页 / 问答页的每段 AI 输出旁有一个"📎 展开召回"小按钮。点击后弹出：

- **本次召回的片段**（N 个，按相似度排序）：每个含文件名 / 位置标签 / 文本预览 / 相似度
- **潜在漏读 tab**：相似度刚刚低于阈值的片段（让用户判断要不要追问）
- **右键操作**：用户可"强制纳入下一次回答"——给用户"召回总指挥"的掌控感

#### 验收标准

- [ ] 所有 AI 输出页都有"展开召回"按钮
- [ ] 召回详情面板含相似度、文件名、位置标签、文本预览
- [ ] "潜在漏读"列表能正常显示
- [ ] 用户能强制纳入某个片段并触发重新生成

### F8 · 大文档切片导出包 【P2 | M3】

#### 用户故事

> 作为 NoteCapt 用户，我想把一份 280 页的 PDF 喂给 ChatGPT，但 ChatGPT 的上下文装不下——我希望 NoteCapt 帮我按章节切成多份小文件，命名规范让 ChatGPT 排序不乱，每份都自带"反幻觉前言"。

#### 功能描述

用户右键大文档 → "导出为切片包"。生成一个文件夹：

```
保险保障一生_切片包/
├── _00_index.md                       入口 + 反幻觉前言
├── _01_前言_p1-p4.md
├── _02_第一章_何为保险_p5-p12.md
├── _03_第一章_保险的本质_p13-p22.md
├── ...
└── _99_glossary.md                    术语表
```

**命名规则**（详见物理形态规范 §8.3）：
- 前置下划线 + 补零序号 → 字典序排序不乱
- 主题词在中间 → 人能读懂
- 页码区间在末尾 → 精确定位

**切片包是只读副本**——不进 NoteCapt 主数据库，避免数据双源。

#### 验收标准

- [ ] 右键菜单有"导出为切片包"
- [ ] 用户可选切片粒度（按章节 / 按 50 页 / 按 100 页）
- [ ] 导出文件命名严格遵守排序规则
- [ ] 每片 + 入口文件都带反幻觉前言
- [ ] 切片包在 ChatGPT / Claude 中能被正确识别为有序系列

### F9 · 分类系统升级（多标签 + 用户自定义）【P1 | M3-M4】

#### 用户故事

> 作为 NoteCapt 用户，一份保险 PDF 对我来说**同时**是"项目/2026 人生保险规划"的素材 + "资源/金融通识"的资料——我希望分类系统能容忍这种"同时属于多个类别"，不要强迫我二选一。

#### 功能描述

**分类规则**：从"PARA 四桶硬编码"升级为"用户的 AI Prompt 决定分类"：
- AI Prompt 配置面板的 classify prompt 已经存在（截图 1）
- 把它的输出 schema 从单分类改为 **多分类数组**
- PARA 只是默认 prompt，用户可改

**数据模型**：
- 一个文件可以同时属于多个分类
- "项目 / 课程"是**查看视角**（view），不是物理容器
- 检索默认全局；用户可在搜索框加 scope 限定

**UI**：
- 标签条不再是"PARA 四桶单选"，而是"多分类并存 + 用户标签"
- 知识库 / 工作区可按多种维度切视图

#### 验收标准

- [ ] classify prompt 输出 schema 支持多分类
- [ ] 一个文件能在多个分类下同时出现
- [ ] 跨容器检索默认可用，无需开关
- [ ] 用户能自定义新分类维度（"考研重点"、"工作参考"）

#### 迁移风险

- 现有用户的"单分类"数据需要迁移脚本
- 迁移规则：原 PARA 单分类作为多分类的"主分类"保留，confidence 设为现有值

### F10 · 表格双重表示 【P1 | M2】

#### 用户故事

> 作为 NoteCapt 用户，我导入的文档里经常含表格（金融报告、对比表、参数表）——我希望 AI 能像理解段落一样准确理解表格内容，而不是看到表格就跳过。

#### 功能描述

NoteCapt 在转 Markdown 时，对表格做**双重表示**：
- 保留原表格的 markdown 形态（用户阅读用）
- 在表格下方自动生成"每行一句自然语言"的折叠区域（AI 检索用）

```markdown
| 产品 | 价格 | 保额 |
| --- | --- | --- |
| 保险 A | 200元 | 10万 |
| 保险 B | 300元 | 15万 |

<details class="notecapt-table-nl">
<summary>📊 表格语义展开（供 AI 检索）</summary>

- 保险 A：价格 200 元，保额 10 万
- 保险 B：价格 300 元，保额 15 万

</details>
```

**召回时**：表格 + 每行都进入检索池。表格命中 → 一起返回完整表格作为上下文。

#### 验收标准

- [ ] PDF / DOCX 中的表格自动双重表示
- [ ] 表格语义展开默认折叠
- [ ] 表格查询的准确率（"保险 A 价格多少"）比单表格表示提升 ≥ 30%

### 5.1 流程 A：拖入一份新 PDF

```
用户拖入 280 页 PDF

  ↓ 后台流水线（数分钟）

[3] 提取 PDF 自带高亮 / 批注（F5）

```

### 5.2 流程 B：录一段课堂讲座

```

  ↓

[2] 后台转录
[3] VAD + 主题转折切分成语义段（F1）
[4] 低置信度词标记

[6] 更新 Master Index
[7] 反幻觉前言注入

[9] 生成 _timeline_YYYY-MM-DD.md（F6）
```

### 5.3 流程 C：镜子反馈

```
用户在知识单元点"和 AI 核对一下"
  ↓
NoteCapt 拼装 prompt：
  - 注入反幻觉前言（F2）
  - 注入 Master Index 摘要（F3）
  - 注入 Glossary（F4）
  - 注入用户输入的解释
  ↓
LLM 召回：
  - 语义召回（M2 后引入向量）
  - 关键词召回
  - Glossary 锚词召回
  - 用户标注片段权重 boost（F5）
  - 时间窗内跨模态联合召回（F6，如果适用）
  ↓
LLM 输出反馈：
  - "你抓住了 A、B"（每条挂位置标签 F1）
  - "你漏了 C（在 ⟨p.12⟩，你曾在此高亮过 F5）"
  - "你讲偏了 D"
  ↓
用户看到反馈后可"展开召回"看 AI 用了哪些片段（F7）
```

### 5.4 流程 D：导出切片包给 ChatGPT

```
用户右键 280 页 PDF → "导出为切片包"

弹窗：选择切片粒度（按章节 / 按页数）
  ↓

  - 8 个切片 .md（每片含反幻觉前言 + 子级 TOC + 回链）
  - 1 个 _00_index.md（含完整导览）
  - 1 个 _99_glossary.md
  ↓
用户拖整个文件夹到 ChatGPT
  ↓
ChatGPT 按字典序读入，反幻觉前言立即生效
```
