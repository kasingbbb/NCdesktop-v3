# 产品大师 — 部署说明

## 1. 文件清单

```
product-master/
├── SKILL.md              # Controller 调度逻辑（必须）
├── lead-pm-prompt.md     # 主力产品经理子 Agent prompt（必须）
├── reviewer-prompt.md    # 纠错大师子 Agent prompt（必须）
├── watchdog.sh           # 看门狗脚本（推荐，防卡死）
└── README.md             # 项目说明
```

## 2. 部署步骤

### 步骤 1：复制文件

将整个 `product-master/` 文件夹放到 OpenClaw 的 skills 目录：

```bash
cp -r product-master ~/.openclaw/skills/
```

### 步骤 2：配置 Web Search（推荐）

编辑 `~/.openclaw/openclaw.json`，添加：

```json5
{
  tools: {
    web: {
      search: {
        provider: "duckduckgo"   // 免费，免 API Key
      }
    }
  }
}
```

然后启用 DuckDuckGo 插件：

```bash
openclaw plugins enable duckduckgo
```

### 步骤 3：启动看门狗（推荐）

```bash
nohup bash ~/.openclaw/skills/product-master/watchdog.sh > /tmp/watchdog.log 2>&1 &
```

看门狗自动检测子 Agent 超时，5 分钟无响应发唤醒，10 分钟发收敛。

### 步骤 4：重启 Gateway

```bash
openclaw gateway run
```

## 3. 使用方式

在对话中说：

> 启用产品大师，{你的任务描述}

产品大师会自动走三阶段流程：方案共识 → 交互验证 → 文档定稿。每个阶段结束后等你确认。

## 4. 用户指令

| 指令 | 效果 |
|------|------|
| `到哪了` | 查看当前进度 |
| `收敛` | 跳过当前步骤，进入下一步 |
| `重试` | 重新派发当前子 Agent |
| `跳过` | 标记当前步骤完成 |

## 5. 常见问题

**Q: 子 Agent 长时间没返回？**
A: 看门狗会自动处理。也可以说「到哪了」查看进度，「收敛」强制进入下一步。

**Q: 阶段二 Demo 没按预期嵌入参考模板？**
A: 确认任务描述中提到了参考 Demo 的文件路径。

**Q: 支持什么类型的 PRD？**
A: 功能型（完整流程）、策略型（加埋点实验）、修复型（简化阶段一二）、架构型（跳过 Demo）。
