## 一、产品定位与范围

### 1.1 产品在生态中的位置

```
┌─────────────────────────────────────────────────────────────────┐
│                  Omni / Arca 产品生态                             │
│                                                                 │
│   物理世界                                                       │
│   (板书/纸书/语音)                                               │
│        │                                                        │
│        ▼                                                        │
│   ┌──────────┐    TF 卡     ┌───────────────┐    API/拖拽       │
│   │ 硬件终端  │ ──────────▶ │  NCdesktop    │ ──────────▶      │
│   │ (Arca)   │ ◀────────── │  桌面软件 ★   │    NotebookLM    │
│   │          │   双向同步    │  (本文档范围)  │    等大模型      │
│   └──────────┘              └───────────────┘                   │
│   端侧 AI 预处理              非线性剪辑台                        │
│   · 拍照/扫描/录音            · 时空记忆轴                        │
│   · 自动打标分类              · 全局悬浮窗                        │
│   · 实况知识绑定              · LLM 直连桥                        │
│                              · 桌面端采集                        │
└─────────────────────────────────────────────────────────────────┘
```

**NCdesktop 的核心使命**：作为硬件 TF 卡与云端大模型之间的**"知识非线性剪辑台"**。它不是一个普通的笔记软件，而是一个**多模态知识资产的可视化管理与输出中枢**。

### 1.2 一句话定义

> 从硬件 TF 卡读取 AI 预处理的多模态知识碎片（音频、照片、扫描文本），在时空记忆轴上进行非线性组织，最终输出为大模型可消费的高纯度结构化数据。

### 1.3 核心差异点

| 维度 | 传统笔记软件 | NCdesktop |
|------|-------------|-----------|
| 数据来源 | 用户手动输入 | 硬件自动采集 + 桌面拖拽双通道 |
| 组织方式 | 文件夹/标签的平面树 | **时间轴为骨架的多模态关联图** |
| 核心交互 | 编辑器 | **类视频剪辑轨道** |
| 输出目标 | 导出文件 | **一键喂给 AI 大模型** |
| 隐私模型 | 云端存储 | **默认离线，本地优先** |

## 二、目标用户与使用场景

### 2.1 用户画像

#### P0 用户：留学生 / 硬核学霸（张同学）

| 属性 | 描述 |
|------|------|
| 年龄 | 20-28 岁 |
| 设备 | MacBook Pro + Arca 硬件 |
| 核心痛点 | 英文专业课听不懂、录音和板书对不上、实体书标注分散 |
| 期望 | 课后打开软件，一条时间轴回放整堂课，音画完美同步 |
| 付费意愿 | 高（已购买 Arca 硬件） |

**典型场景**：
> 张同学上完一堂 90 分钟的量子力学课。课上用 Arca 拍了 12 张板书、扫描了教材 3 页、全程录音。回到宿舍，TF 卡一插，NCdesktop 自动呈现一条从 09:00 到 10:30 的时间轴。12 张板书精准悬挂在对应的录音时间点上。她点击第 7 张板书（薛定谔方程推导），录音自动倒退 5 秒开始播放教授当时的讲解。她把这整段时间轴块拖进 NotebookLM，AI 立即生成了完整的课堂笔记。

#### P1 用户：高阶知识工作者（李律师）

| 属性 | 描述 |
|------|------|
| 年龄 | 30-45 岁 |
| 设备 | MacBook Air + Arca 硬件 |
| 核心痛点 | 会议录音整理耗时、卷宗信息分散、对云端存储有隐私焦虑 |
| 期望 | 会后 5 分钟出结构化纪要，所有数据不离开本机 |
| 付费意愿 | 极高（效率 = 金钱） |

**典型场景**：
> 李律师参加客户会议。Arca 全程录音 45 分钟，期间拍了 3 张白板照片。会后插入 TF 卡，NCdesktop 自动展开时间轴，照片精准吸附在讨论到对应议题的时间点。他选中整个时间轴，一键导出为 Markdown，再发给 NotebookLM 生成结构化会议纪要。全程数据未离开本机。

#### P2 用户：信息收集极客（王研究员）

| 属性 | 描述 |
|------|------|
| 年龄 | 25-40 岁 |
| 设备 | Mac Studio + Arca 硬件 |
| 核心痛点 | 每天在手机、浏览器、PDF 之间来回倒腾信息，资料最终"吃灰" |
| 期望 | 一个统一入口把所有信息源汇聚、自动分类、随时喂 AI |
| 付费意愿 | 中高 |

**典型场景**：
> 王研究员在写论文综述。他把 15 篇 PDF 从 Finder 拖入 NCdesktop 全局悬浮窗，软件自动提取标题和关键词，归入"量子计算/综述"文件夹。同时他把 Arca 硬件采集的 3 段学术讲座录音也导入。所有资料在一个统一的知识库中，按主题自动聚合。

### 2.2 使用场景矩阵

## 三、信息架构

### 3.1 核心概念模型

```
┌───────────────────────────────────────────────────────────────┐
│                      NCdesktop 概念模型                        │
│                                                               │
│   知识库 (Library)                                             │
│   └── 项目 (Project)  ← 一堂课 / 一个会议 / 一个研究主题       │
│       ├── 时间轴 (Timeline)  ← 有录音时自动生成                 │
│       │   ├── 音频轨道 (AudioTrack)  ← 录音波形                 │
│       │   ├── 关键帧 (Keyframe)  ← 照片/扫描在时间点上的锚定    │
│       │   └── 标记 (Marker)  ← 用户手动添加的时间标记           │
│       ├── 素材 (Asset)  ← 照片、扫描文本、PDF、网页等           │
│       │   ├── 照片 (Photo)                                     │
│       │   ├── 扫描文本 (ScanText)                              │
│       │   ├── 文档 (Document)  ← PDF、网页、Markdown            │
│       │   └── 音频片段 (AudioClip)  ← 实况知识的 20s 音频       │
│       ├── AI 标签 (Tag)  ← 端侧/桌面端 AI 自动生成             │
│       └── 笔记 (Note)  ← 用户手写的附加笔记                    │
│                                                               │
│   全局悬浮窗 (Dropzone)  ← 独立于主窗口的常驻入口               │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

### 3.3 完整数据模型

```typescript
// ===== 知识库 =====
interface Library {
  id: string;                   // UUID
  name: string;                 // "我的知识库"（默认唯一）
  rootPath: string;             // 本地存储根路径
  createdAt: string;            // ISO 8601
}

// ===== 项目（对应一堂课 / 一个会议 / 一个研究主题）=====
interface Project {
  id: string;
  libraryId: string;
  name: string;                 // AI 自动命名或用户自定义
  description: string;          // 项目描述
  coverAssetId: string | null;  // 封面素材 ID
  source: ProjectSource;        // 来源
  tags: Tag[];                  // AI + 手动标签
  isPinned: boolean;
  isArchived: boolean;
  createdAt: string;
  updatedAt: string;
  metadata: ProjectMetadata;
}

type ProjectSource =
  | { type: "tf_card"; deviceId: string; sessionId: string }  // 来自硬件
  | { type: "dropzone" }                                       // 来自桌面拖拽
  | { type: "manual" };                                        // 手动创建

interface ProjectMetadata {
  totalDuration: number | null;   // 总录音时长（秒），无录音为 null
  assetCount: number;             // 素材总数
  wordCount: number;              // 文字总量（OCR + 扫描 + 笔记）
  importedAt: string | null;      // TF 卡导入时间
}

// ===== 时间轴 =====
interface Timeline {
  id: string;
  projectId: string;
  startTime: string;              // 录音开始的绝对时间 ISO 8601
  endTime: string;                // 录音结束的绝对时间
  duration: number;               // 总时长（秒）
  audioTracks: AudioTrack[];
  keyframes: Keyframe[];
  markers: Marker[];
}

// ===== 音频轨道 =====
interface AudioTrack {
  id: string;
  timelineId: string;
  filePath: string;               // 音频文件本地路径
  fileName: string;
  format: "wav" | "m4a" | "mp3" | "aac";
  duration: number;               // 时长（秒）
  sampleRate: number;             // 采样率
  channels: number;               // 声道数
  waveformData: string;           // 波形数据文件路径（预渲染）
  transcription: Transcription | null;  // AI 转录结果
  offsetInTimeline: number;       // 在时间轴中的起始偏移（秒）
}

interface Transcription {
  id: string;
  audioTrackId: string;
  language: string;               // "en" | "zh" | "ja" 等
  segments: TranscriptionSegment[];
  status: "pending" | "processing" | "completed" | "failed";
}

interface TranscriptionSegment {
  startTime: number;              // 相对音频起始的秒数
  endTime: number;
  text: string;
  confidence: number;             // 0-1
  speaker: string | null;         // 说话人标识（如可识别）
}

// ===== 关键帧（素材在时间轴上的锚定）=====
interface Keyframe {
  id: string;
  timelineId: string;
  assetId: string;                // 关联的素材
  anchorTime: number;             // 在时间轴上的锚定时间（秒）
  liveAudioClipId: string | null; // 实况知识音频片段（前10s+后10s）
  source: "auto" | "manual";     // 自动吸附还是手动放置
}

// ===== 时间标记 =====
interface Marker {
  id: string;
  timelineId: string;
  time: number;                   // 在时间轴上的时间（秒）
  label: string;                  // 标记文字
  color: string;                  // 标记颜色
  type: "bookmark" | "chapter" | "important" | "question";
}

// ===== 素材（所有类型的知识碎片）=====
interface Asset {
  id: string;
  projectId: string;
  type: AssetType;
  name: string;                   // AI 自动命名或原始文件名
  filePath: string;               // 本地文件路径
  fileSize: number;               // 字节
  mimeType: string;
  tags: Tag[];
  capturedAt: string;             // 采集时间（硬件端记录）
  importedAt: string;             // 导入到软件的时间
  source: AssetSource;
  aiAnalysis: AIAnalysis | null;  // AI 分析结果
  isStarred: boolean;
}

type AssetType =
  | "photo"                       // 拍照
  | "scan_text"                   // 扫描笔文本
  | "audio_clip"                  // 实况知识音频片段
  | "pdf"                         // PDF 文档
  | "webpage"                     // 网页快照
  | "markdown"                    // Markdown 文件
  | "image"                       // 通用图片（非拍照）
  | "other";

type AssetSource =
  | { type: "tf_card_camera" }    // 硬件摄像头
  | { type: "tf_card_scanner" }   // 硬件扫描笔
  | { type: "tf_card_mic" }       // 硬件麦克风（实况知识片段）
  | { type: "dropzone_drag" }     // 桌面拖拽
  | { type: "dropzone_paste" }    // 桌面粘贴
  | { type: "manual_import" };    // 手动导入

interface AIAnalysis {
  summary: string;                // AI 内容摘要
  topics: string[];               // AI 识别的主题
  ocrText: string | null;         // OCR 提取的文字（图片类型）
  language: string;               // 检测到的语言
  suggestedTags: string[];        // AI 建议的标签
  suggestedName: string;          // AI 建议的名称
}

// ===== 标签 =====
interface Tag {
  id: string;
  name: string;
  color: string;
  source: "ai" | "user";         // AI 生成还是用户手动创建
  usageCount: number;             // 使用次数
}

// ===== 用户笔记 =====
interface Note {
  id: string;
  projectId: string;
  assetId: string | null;         // 关联的素材（可选，null 为项目级笔记）
  timelineTime: number | null;    // 关联的时间轴时间点（可选）
  content: string;                // Markdown 格式
  createdAt: string;
  updatedAt: string;
}

// ===== 导出配置 =====
interface ExportConfig {
  format: "markdown" | "pdf" | "html" | "json";
  includeAudioTranscription: boolean;
  includeOCRText: boolean;
  includePhotos: boolean;
  includeNotes: boolean;
  includeTimestamps: boolean;
  timeRange: { start: number; end: number } | null;  // null = 全部
}

// ===== 应用设置 =====
interface AppSettings {
  // 外观
  theme: "light" | "dark" | "system";
  sidebarWidth: number;
  timelineHeight: number;

  // TF 卡
  autoImportOnConnect: boolean;     // TF 卡插入时自动导入
  importDeleteOriginal: boolean;    // 导入后删除 TF 卡原文件
  defaultImportPath: string;        // 默认导入路径

  // 悬浮窗
  dropzoneEnabled: boolean;         // 悬浮窗是否启用
  dropzonePosition: { x: number; y: number };
  dropzoneSize: "small" | "medium" | "large";
  dropzoneAutoClassify: boolean;    // 拖入后自动 AI 分类

  // 音频
  defaultPlaybackSpeed: number;     // 默认播放速度
  preRollSeconds: number;           // "由图寻音"倒退秒数（默认 5）
  waveformColor: string;

  // AI
  transcriptionLanguage: string;    // 默认转录语言
  aiClassificationEnabled: boolean; // AI 自动分类开关
  llmBridgeTarget: LLMTarget;      // 默认 LLM 目标

  // 隐私
  analyticsEnabled: boolean;        // 数据分析（默认关闭）
  dataStoragePath: string;          // 数据存储路径
}

type LLMTarget =
  | { type: "notebookLM" }
  | { type: "chatgpt" }
  | { type: "claude" }
  | { type: "custom"; endpoint: string };
```

## 四、核心功能详细规格

### F4. 全局悬浮窗 (Global Dropzone)

#### 4.1 功能概述

常驻桌面的半透明悬浮窗口，作为 NCdesktop 在主窗口之外的**"信息入口"**。类似 macOS 垃圾桶的直觉式操作，用户可以将任何信息碎片（文件、链接、截图、文本）拖入其中。

#### 4.2 悬浮窗状态机

```
                    ┌──────────────┐
                    │   隐藏状态    │  ← 用户主动隐藏
                    │  (Hidden)    │
                    └──────┬───────┘
                           │ 菜单栏图标点击 / 快捷键
                           ▼
                    ┌──────────────┐
           ┌───────│   待机状态    │───────┐
           │       │  (Idle)      │       │
           │       └──────┬───────┘       │
           │              │               │
      点击展开        检测到拖拽       右键菜单
           │              │               │
           ▼              ▼               ▼
    ┌────────────┐ ┌────────────┐ ┌────────────┐
    │  展开状态   │ │  吸引状态   │ │  菜单状态   │
    │ (Expanded) │ │ (Attract)  │ │  (Menu)    │
    │            │ │ 放大+发光   │ │            │
    │ 显示最近   │ │ 引导拖放   │ │ 设置/隐藏  │
    │ 导入记录   │ │            │ │ 打开主窗口  │
    └────────────┘ └─────┬──────┘ └────────────┘
                         │
                    文件放入
                         │
                         ▼
                  ┌────────────┐
                  │  处理状态   │
                  │(Processing)│
                  │ AI 分类中   │
                  │ 进度动画    │
                  └─────┬──────┘
                        │
                   处理完成
                        │
                        ▼
                  ┌────────────┐
                  │  完成状态   │
                  │ (Complete) │
                  │ ✓ 归档成功  │
                  │ 2秒后回到   │
                  │ Idle       │
                  └────────────┘
```

#### 4.3 交互规格

| 交互 | 行为 |
|------|------|
| **拖拽文件到悬浮窗** | 窗口放大 1.2x + Liquid Glass 光晕动画 → 松手后缩回 → 开始 AI 处理 |
| **支持的拖拽类型** | 文件（PDF/图片/音频/视频/Markdown）、文件夹、网页链接、纯文本、截图 |
| **单击悬浮窗** | 展开显示最近 5 条导入记录的卡片列表 |
| **双击悬浮窗** | 打开 NCdesktop 主窗口 |
| **右键悬浮窗** | 弹出快捷菜单：打开主窗口 / 设置 / 暂停自动分类 / 隐藏 |
| **拖拽移动** | 按住悬浮窗边缘可在桌面任意位置拖拽放置 |
| **全局快捷键** | `⌘⇧D` 显示/隐藏悬浮窗 |
| **多文件拖入** | 支持批量拖入，显示进度"3/15 已处理"，失败项单独标红 |

#### 4.4 AI 自动分类逻辑

```
输入素材
    │
    ▼
┌──────────────────┐
│  1. 文件类型识别   │  → MIME type 检测
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  2. 内容提取       │  → PDF: 提取文本 / 图片: OCR / 网页: 抓取正文
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  3. 主题分析       │  → 关键词提取 + 分类模型
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  4. 匹配现有项目   │  → 与已有 Project 的主题相似度 > 0.7 则归入

└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  5. 自动命名       │  → 基于内容生成简洁中文/英文名称
│     自动打标       │  → 生成 2-5 个标签
└──────────────────┘
```

| 交互 | 行为 |
|------|------|

| 手动添加关键帧 | 从素材列表拖拽素材到时间轴的某个时间点 |

**双向联动（Magic Moment）：**

| 模式 | 触发 | 动画效果 |
|------|------|----------|
| **由图寻音** | 点击任意关键帧照片/素材 | 播放头平滑动画滑动到对应时间点 → 倒退 `preRollSeconds`（默认 5 秒）→ 自动开始播放 → 照片在上方高亮放大 |

### F6. LLM 直连桥 (The LLM Bridge)

#### 6.1 功能概述

将经过时间轴整理的知识数据，一键输送给大模型（NotebookLM、ChatGPT、Claude 等），终结"Garbage In, Garbage Out"。

#### 6.2 导出数据格式

当用户选择"发送到 LLM"时，系统自动将选中的内容组装为结构化 Markdown：

```markdown
# 量子力学 Lecture 07 — 薛定谔方程
> 录制时间：2026-03-20 09:00-10:30
> 来源：课堂录音 + 板书拍摄
> 素材数量：12 张板书 + 3 页教材扫描 + 90 分钟录音

---

## 时间线

### [09:05] 课程开场
**转录**：Today we're going to derive the time-dependent Schrödinger equation...

### [09:14] 板书：波函数定义
**[照片: wave-function-definition.jpg]**
**OCR 文字**：ψ(x,t) = A·e^(i(kx-ωt))
**转录**：The wave function psi describes the quantum state of a particle...

### [09:32] 扫描：教材第 47 页
**[扫描: textbook-p47.txt]**
**内容**：The probability of finding the particle between x and x+dx is |ψ|²dx...

... (更多时间线条目)

---

## 附加笔记
- 薛定谔方程的推导需要用到德布罗意关系
- 教授强调边界条件的重要性

## 标签
#量子力学 #薛定谔方程 #波函数 #物理
```

#### 6.3 LLM 连接方式

| 目标 | 连接方式 | 操作 |
|------|----------|------|
| **NotebookLM** | 浏览器拖拽 | 导出 Markdown 文件 → 打开 NotebookLM 网页 → 用户手动上传/拖入 |
| **ChatGPT** | API 直连（可选）/ 剪贴板 | 将结构化内容复制到剪贴板 / 通过 API Key 直接发送 |
| **Claude** | API 直连（可选）/ 剪贴板 | 同上 |
| **自定义** | Webhook / API | 用户配置自定义 endpoint |

#### 6.4 交互流程

```
用户操作                                     系统响应
────────                                     ────────

1. 选择导出范围
   ├─ 整个项目                               组装全部数据
   ├─ 时间轴选区（拖拽选中一段）               组装选区内数据
   └─ 单个素材 + 关联上下文                   组装素材 ± 前后 30s 转录

2. 点击"发送到 LLM"按钮
   └─ 或拖拽到 LLM 目标图标                   弹出导出预览面板

3. 预览面板
   ├─ 预览生成的 Markdown                    实时渲染
   ├─ 勾选/取消要包含的内容类型               动态更新预览
   │   ☑ 音频转录文本
   │   ☑ OCR 提取文字
   │   ☑ 照片（作为附件）
   │   ☑ 用户笔记
   │   ☑ 时间戳
   └─ 选择目标 LLM                           显示可用目标列表

4. 确认发送
   ├─ NotebookLM: 导出 .md 文件 + 自动打开浏览器
   ├─ API 目标: 显示发送进度 → 成功/失败提示
   └─ 剪贴板: 复制到剪贴板 + Toast 提示
```

```
│   ├── 2026-03-20_0900_lecture/  # 一个会话 = 一次连续使用
│   │   ├── session.json          # 会话元数据（时间、时长、AI 分类结果）
│   │   ├── audio/
```

## 六、非功能性需求

### 6.1 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 冷启动 | < 500ms | Tauri + Vite 优化 |
| TF 卡导入速度 | > 50MB/s | Rust 端并行文件复制 |
| 时间轴首次加载 | < 1s | 90 分钟录音 + 50 个关键帧 |
| 波形渲染 | < 200ms | 使用预渲染波形数据 |
| 搜索响应 | < 200ms | SQLite FTS5 |
| 内存占用（浏览） | < 150MB | 含加载一个 90min 项目 |
| 内存占用（空闲） | < 80MB | 无项目打开时 |
| 音频播放延迟 | < 50ms | 点击到出声 |
| 动画帧率 | 60fps | 时间轴拖拽、Liquid Glass 动效 |

### 6.2 隐私与安全

| 原则 | 实现 |
|------|------|
| **默认离线** | 所有数据本地存储，无需网络即可使用全部核心功能 |
| **零遥测** | 不收集任何用户数据，analyticsEnabled 默认 false |
| **本地 AI** | 音频转录和分类使用本地模型（Whisper.cpp / Apple MLX） |
| **可选联网** | 仅 LLM 直连桥功能需要网络（用户主动触发） |
| **加密存储** | 用户数据目录支持 FileVault 加密（跟随系统） |
| **TF 卡安全** | 导入后可选安全擦除 TF 卡上的原始数据 |

- 所有交互支持键盘操作
- 时间轴支持键盘导航（方向键、Tab 键在关键帧间跳转）
- 尊重系统"减少动效"和"降低透明度"设置
- 音频播放支持字幕/转录文本显示

## 附录 A：术语表

| 术语 | 定义 |
|------|------|
| **会话 (Session)** | 硬件端一次连续使用的采集单元，对应软件中的一个 Project |
| **实况知识 (Live Knowledge)** | 拍照/扫描瞬间自动截取的前10s+后10s音频，绑定在素材上 |
| **关键帧 (Keyframe)** | 素材（照片/扫描文本）在时间轴上的锚定点 |
| **Magic Moment** | "由图寻音"和"随音现图"双向联动交互的统称 |
| **Pre-roll** | 由图寻音时，录音在跳转位置前倒退的秒数（默认 5 秒） |
| **Dropzone** | 全局悬浮窗的技术名称 |
| **LLM Bridge** | 将知识数据导出为大模型可消费格式的功能模块 |
| **端侧 AI** | 运行在 Arca 硬件 NPU 上的本地 AI 模型 |
| **TF 卡** | 硬件使用的 TF（MicroSD）存储卡 |
